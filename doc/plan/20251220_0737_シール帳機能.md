# 概要
ユーザーがあつめた景品をシールとしてキャンパス（シール帳）に配置し、他ユーザーも共有できる機能。

# 要件
キャンバス（シール帳）のサイズはスマホの画面程度のサイズ。全ユーザー固定。
配置したシールはポジションなどを保存し、再アクセスしたときも同じ位置に表示される。
位置などのデータはすべてサーバー側に保存する。
保存ボタンを押すことでサーバーへ保存する。
初期表示時にはサーバーから情報を取得して画面に描画する。
表示モードと編集モードの２つを用意する。
表示モードでは、表示のみに特化しており、編集はできない。
編集モードでは、シールの新規配置、位置変更、削除が可能。
位置変更ではシールの向きを回転することができる。拡縮もできる。

# トレード制約
シール帳に貼ってある景品は、トレードすることはできない。
なので、トレードの際にシール帳の景品は除外する。（トレード対象として選択できない、表示されない）
逆に、トレード申請中のものはシール帳に新たに貼ることはできない。

# 画面遷移
トップ画面の一番したに、「シール帳」のボタンを新たに登録する。

# シール帳画面構成
- キャンパス（シール帳）
    - 各シール（シール帳に貼ってある景品）
    - 各シールの編集ボタン
    - 各シールの削除ボタン
- 保存ボタン
- 編集モード切り替えボタン
- 戻るボタン（保存していない変更がある場合はモーダルで確認を行う）
